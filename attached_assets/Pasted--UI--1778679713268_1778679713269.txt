لا تقم بمراجعة جميع الملفات أو بنية النظام.
نفّذ التعديل على مستوى الواجهة (UI) وصلاحيات المستخدم فقط.

المطلوب:
إنشاء نظام صلاحيات متقدم لصفحة الإعدادات يسمح بالتحكم في:
- ظهور كل قسم (Show / Hide)
- وإمكانية التعديل داخله (Edit / Read-only)

الأقسام المستهدفة من صفحة الإعدادات:

1) اختيار البيئة (Production / Test)
2) التكامل الخارجي (API Integration)
3) بيانات اعتماد نظام رصد (DTTS)

--------------------------------------

أولًا: إضافة الصلاحيات في إدارة المستخدمين

إضافة صلاحية جديدة باسم:
"إدارة إعدادات التكامل"

داخل هذه الصلاحية:

لكل قسم من الأقسام الثلاثة يتم إضافة خيارين (Checkboxes):

- عرض (Show)
- تعديل (Edit)

--------------------------------------

ثانيًا: منطق التحكم في الواجهة

1) إذا كان خيار "عرض" غير مفعّل:
- يتم إخفاء القسم بالكامل من واجهة المستخدم.

⚠️ مهم جدًا:
- إخفاء القسم لا يعني إيقاف الخدمة.
- جميع العمليات المرتبطة بهذا القسم تظل تعمل بشكل طبيعي في الخلفية.

----------------------

2) إذا كان "عرض" مفعّل و "تعديل" غير مفعّل:
- يتم عرض القسم للمستخدم
- جميع الحقول تكون Read-only أو Disabled
- يتم تعطيل جميع الأزرار داخل القسم

تنفيذ Read-only mode:

- تعطيل الحقول:
  - Base URL
  - Username
  - Password
  - اختيار البيئة
  - مفاتيح API

- تعطيل الأزرار:
  - حفظ الإعدادات
  - اختبار الاتصال
  - إنشاء مفتاح
  - حذف مفتاح

- المظهر:
  - لون باهت (greyed out)
  - cursor: not-allowed
  - منع الكتابة والتفاعل

- إظهار رسالة:
  "غير مسموح لك بالتعديل، يرجى التواصل مع الإدارة"

----------------------

3) إذا كان "عرض" و "تعديل" مفعّلين:
- يعمل القسم بشكل طبيعي (عرض + تعديل)

--------------------------------------

ثالثًا: إلغاء كلمة المرور

- إزالة طلب كلمة المرور نهائيًا من صفحة الإعدادات.
- لا يتم طلب Password لأي مستخدم (Admin أو User).
- الدخول يتم مباشرة.

--------------------------------------

رابعًا: ملاحظات مهمة جدًا

- إخفاء الأقسام هو إخفاء واجهة فقط (UI Visibility).
- لا يتم إيقاف:
  - Integration
  - API
  - Environment
  - أو أي خدمة في الخلفية

- جميع العمليات في الخلفية تظل تعمل حتى لو القسم مخفي.

- Admin يملك التحكم الكامل في:
  - إظهار/إخفاء الأقسام
  - السماح أو منع التعديل

- لا يتم تعديل أي API أو Backend Logic.
- التعديل مقتصر على واجهة المستخدم (UI فقط).
